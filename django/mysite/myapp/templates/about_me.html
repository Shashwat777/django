    Shell Implementation: Implemented a basic shell. Our shell implementation did not execute the existing shell program. We build these functionalities by ourselves (using fork, exec, wait, etc.). By default, the shell program waits for user input from the stdin. After the user enters some command, the shell program parsed the input to interpret I/O redirection, pipe, etc. After, interpreting the command as described the shell program again waits for user input until the user enters the exit command. The program even handled nested commands like “cat /bin/ls | sort | uniq | wc -l 2>&1 1>output.txt”.

    Implementing Demand paging in xv6: The goal of this assignment was to understand the paging subsystem of an operating system. The first component was to implement is demand paging. Demand paging lazily maps a physical page corresponding to a virtual page during the time of dereferencing. In addition to demand paging, implemented swapping. If the system runs out of physical memory, then the swapping subsystem reuse an already used physical page by saving its contents to secondary storage (e.g., disk).
